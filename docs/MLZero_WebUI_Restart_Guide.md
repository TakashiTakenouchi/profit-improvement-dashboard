# MLZero Web UI 再起動ガイド

## Web UIを再起動する必要がある場合

### ✅ 再起動が**不要**な場合

1. **タスクが完了または失敗した後**
   - 新しいタスクを実行する場合は、そのまま実行できます
   - Web UIを再起動する必要はありません

2. **手動でコードを実行した後**
   - 外部でコードを実行しても、Web UIに影響はありません
   - そのまま新しいタスクを実行できます

3. **依存関係をインストールした後**
   - システム環境の変更は、Web UIの再起動を必要としません

### ❌ 再起動が**必要**な場合

1. **設定ファイルを変更した後**
   - `autogluon_config.yaml`を変更した場合
   - モデル名や`max_tokens`を変更した場合

2. **バックエンドまたはフロントエンドがクラッシュした場合**
   - プロセスが停止している場合

3. **ポートエラーが発生した場合**
   - ポート8509または5000が使用できない場合

## 現在の状況

### 確認方法

```bash
# プロセス確認
wsl bash -c "ps aux | grep -E 'mlzero-backend|mlzero-frontend|streamlit.*8509' | grep -v grep"

# Web UI接続確認
wsl bash -c "curl -s -o /dev/null -w '%{http_code}' http://localhost:8509"
```

### 正常な状態

- バックエンド: 実行中（ポート5000）
- フロントエンド: 実行中（ポート8509）
- HTTPステータス: 200

## 推奨アクション

### 現在の状況（Bashスクリプトを手動実行後）

**✅ Web UIを再起動する必要はありません**

理由：
- バックエンドとフロントエンドは独立して動作します
- 手動でコードを実行しても、Web UIに影響はありません
- 新しいタスクを実行する準備ができています

### 新しいタスクを実行する場合

1. **Web UIでそのまま新しいタスクを実行**
   - ブラウザで`http://localhost:8509`にアクセス
   - 新しいタスクを提出

2. **タスク説明の改善**
   - 依存関係のインストールを明示的に指示：
   ```
   添付ExcelのDailyForecastDataシートを見て90日間の予測を作成してください。
   
   重要: Bashスクリプトの最初に「pip install autogluon.timeseries pandas openpyxl」を追加してからコードを実行してください。
   ```

## トラブルシューティング

### Web UIに接続できない場合

```bash
# プロセスを確認
wsl bash scripts/check_mlzero.sh

# 再起動
wsl bash scripts/restart_mlzero.sh
```

### タスクが実行されない場合

1. バックエンドが正常に動作しているか確認
2. 設定ファイルが正しいか確認
3. 必要に応じて再起動

## まとめ

- ✅ **Web UIを再起動する必要はありません**
- ✅ **そのまま新しいタスクを実行できます**
- ✅ **バックエンドとフロントエンドが正常に動作していれば問題ありません**
